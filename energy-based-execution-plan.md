# Energy-Based Execution Plan

## Purpose

This plan makes the Resilience Proof System realistic by matching work to available capacity.

The system does not require the same output every day. It requires the right level of action for the day.

---

## Low Energy Day

### Goal

Stay connected to the system without forcing a heavy workload.

### Minimum Action

Complete one task that takes five to ten minutes.

### Examples

- Open the assignment and review the prompt
- Write one sentence for a case study
- Add one row to the tracker
- Reply to one LinkedIn comment
- Save one job description for later review
- Add one note to a workflow file
- Set tomorrow's first task

---

## Medium Energy Day

### Goal

Move one important area forward.

### Minimum Action

Complete one school or portfolio task plus one connection action.

### Examples

- Draft 150 words of an assignment
- Build one workflow section
- Comment on one healthcare operations post
- Update one GitHub file
- Research one revenue cycle concept
- Write one LinkedIn post draft

---

## High Energy Day

### Goal

Create proof that can be reused professionally.

### Minimum Action

Complete one deeper school, portfolio, or LinkedIn asset.

### Examples

- Finish a workflow map
- Write a full case study section
- Create a LinkedIn carousel outline
- Build a Google Sheets tracker
- Update the repository README
- Turn coursework into a healthcare operations insight post

---

## Operating Rule

The system succeeds when progress continues without requiring perfect energy, perfect confidence, or perfect timing.
